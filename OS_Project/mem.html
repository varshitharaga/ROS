Contiguous:
In contiguous memory allocation, a process is allocated a contiguous block of memory in the main memory (RAM).
This allocation strategy simplifies memory management as it allows for efficient addressing and retrieval of data.
However, it can lead to fragmentation issues over time, where the available memory becomes fragmented into small, unusable chunks..
Techniques such as compaction or paging are used to mitigate fragmentation and optimize memory utilization in contiguous memory allocation systems. Click Here
Non Contiguous:
In non-contiguous memory allocation, a process is divided into multiple non-contiguous blocks of memory scattered throughout the main memory.
This allocation method is more flexible than contiguous allocation, allowing for better memory utilization as it can fit a process into available free memory blocks regardless of their location.
Fragmentation is less of a concern in non-contiguous allocation since smaller chunks of memory can be used efficiently, reducing wasted space.
However, managing non-contiguous memory requires additional data structures (e.g., page tables in virtual memory systems) to keep track of the location of each memory block, which can incur overhead in terms of memory and processing resources.Click Here
virtual:
Virtual memory allows programs to operate under the illusion of having access to a large, contiguous address space, abstracted from the limitations of physical memory.
Virtual memory systems use a page table or a similar mechanism to map virtual addresses used by programs to physical memory addresses.
Virtual memory systems employ demand paging, where only the portions of a program that are actively being used are loaded into physical memory, while the rest remains on disk. This minimizes the memory footprint of a program and allows for more efficient memory utilization.
Virtual memory provides memory protection mechanisms to prevent unauthorized access to memory regions, enhancing system security and stability.
