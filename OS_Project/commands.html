colcon build --package-select < package-name >

The "colcon build --package-select < package-name >" command is used in ROS (Robot Operating System) to build the specified package and its dependencies using the Colcon build system.

ros2 run < package-name > < none-name >

The command "ros2 run < package-name > < node-name >" is used in ROS 2 to execute a specific node within a package. You would replace "< package-name >" with the name of the ROS 2 package containing the node you want to run, and "< node-name >" with the name of the executable node you wish to run.

ros2 pkg create < package-name >

The "ros2 pkg create < package-name >" command is used to create a new ROS 2 package with the specified name. This command generates the necessary folder structure and files for a basic ROS 2 package, allowing you to start developing your own ROS 2 nodes and components within it.

ros2 doctor

ros2 doctor is a command used to diagnose and provide information about the ROS 2 environment. It checks for common issues and displays the status of various components, helping to ensure that your ROS 2 setup is configured correctly.

ros2 interface

ros2 interface is a command-line tool used to inspect and interact with ROS 2 interfaces. This includes message types, service types, and action types. It allows you to list, show details, and introspect the various interfaces available in your ROS 2 system.
